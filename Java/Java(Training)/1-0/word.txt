
■IT
○IT →インターネットなどの通信とコンピュータを駆使する情報技術の総称。
○ICT →情報通信技術の略で、通信技術を活用したコミュニケーションを意味している。
○IoT →従来インターネットに接続されていなかったモノに通信機能を搭載してインターネットに接続させる技術。

■OS
○OS　→コンピュータを動かすための基本的なソフトウェアのこと。
○Windows　→マイクロソフト社が開発販売しているOS。
○macOS (Mac OS X) →アップル社が開発販売しているMacintosh用のOS
○UNIX →現存する中で最も古いOSで、マルチタスク・マルチユーザーを実現したOSの一種。
○Linux　→世界で最も普及しているオープンソースのOSで、UNIX系OSの一種

■言語・環境
○マークアップ／フロントエンド　
○HTML　→ウェブページの文書構造を作成するために開発されたマークアップ言語。
○CSS →ウェブページのスタイルを指定するための言語。スタイルシート言語の一つ。
○JavaScript　→動的なWebページを作成することのできる、ブラウザ上で動作するプログラミング言語。
○サーバーサイド　
○.NET（C#, VB）→Microsoft社が開発したネットワーク上でアプリケーションを構築する基盤システム
○Java　→オブジェクト指向プログラミング言語の一つで、世界中の基幹システムで用いられている。
 ○Java SE　→javaにおけるAPIをまとめたもので、最も基本的な機能をまとめたもの。
 ○Java EE（Jakarta EE）→javaにおけるAPIをまとめたもので、java SEに拡張機能が加わったもの。
○Perl　→プログラミング言語の一つで、ウェブサービスの開発に適したスクリプト言語。
○PHP　→プログラミング言語の一つで、サーバーサイドのスクリプト言語。WebサービスやWebアプリケーションの開発に使用される。
○Ruby　→日本人が開発した国産のプログラミング言語で、記述量が少ない、オブジェクト指向といった特徴がある。

■サーバーソフト
○Apache　→世界中で使用されているWebサーバーソフトウェア。
○nginx　→フリーかつオープンソースなWebサーバー用のソフトウェア。
○IIS　→Microsoft社が提供しているwindowsのWebサーバーソフトウェア。

■SQL
○SQL　→データベースを操作するための言語の一つで、データベースの定義や操作を行うことができる。
○DB2　→IBM社のリレーショナルデータベース管理システムの製品の総称。
○MySQL →世界で広く使われているデータベース管理システム。
 ○Maria DB　→MySQLから派生したオープンソースのリレーショナルデータベース管理システム。
○Oracle DB　→Oracleが開発・販売しているリレーショナルデータベース管理システム。
○PostgreSQL　→オープンソースのリレーショナルデータベース管理システムである。
○SQLite　→軽量でコンパクトなリレーショナルデータベース管理システムである。
○SQL Server　→Microsoft社が開発・販売しているリレーショナルデータベース管理システムである。
○CRUD　→永続的なデータを取り扱うソフトウェアに要求される４つの基本的な機能であるCreate、Read、Update、Deleteの頭文字を繋げた言葉。
○トランザクション　→複数の処理単位に分割できるものの、分割して扱ってはいけないような一連の処理の流れを一括して扱う方式。
 ○コミット　→トランザクション処理において、関連する処理が全て成功して、結果を確定させること。
 ○ロールバック　→トランザクション処理を全て取り消して最初の時点に戻すこと。
○排他制御（排他ロック）　→データの整合性を保つために、特定のファイルやデータに対するアクセスや更新を制御すること。
 ○楽観ロック　→めったに他者との同時更新は起きないであろうという楽観的な前提の排他制御。
 ○悲観ロック　→他者が同じデータに頻繁に変更を加えるであろうという悲観的な前提の排他制御。
○実行計画　→ユーザが発行したクエリを実行するための計画のこと。
○チューニング　→目的の処理結果を変えずにその処理過程を最適化すること。
○パフォーマンス　→コンピュータの処理速度や実行速度のことを指している。
○スロークエリ　→データベースにおいて時間のかかっているSQLのこと。
○クエリログ　→実行したSQLクエリの履歴が全て記録されていくログのこと。
○IDE（統合開発環境）　→プログラミングをする際に必要なソフトウェアを統合して、同じ画面で操作できるようにしたソフトウェアパッケージのこと。
○Eclipse　→主にjavaの開発環境として使用されるオープンソースの統合開発環境。
○NetBeans　→オラクルを中心としたコミュニティにより開発されているオープンソースの統合開発環境。
○SpringToolSuite　→EclipseをベースにしてSpring Frameworkの開発に必要なものを組み込んだパッケージ。
○DBツール　→データベース管理システムやリレーショナルデータベース管理システム上のデータを容易に扱うためのソフトウェア。
○A5:SQL Mk-2　→データベース開発において生産性を向上させるべく開発されたSQL開発ツール。
○pgAdmin →PostgreSQLをGUIによって管理するツール。
○DBeaver　→様々なデータベースに対応しているSQLクライアントツール。
○MySQL Workbench →MySQL公式の設計・開発・管理を実現する統合GUIツール。
○Sequel Pro　→Mac用のMySQLのデータをGUI操作で管理するためのアプリケーション。
○TeamSQL　→様々なデータベースに対応しているGUIによって管理するためのソフトウェア。

■肩書
○PM　→運営管理や人員確保など、プロジェクトを遂行するのに必要な準備および調達を行う者。
○PL　→損益計算表。一定期間の会社の経営成績を表しており、売り上げとそれにかかった費用、利益が記載されている。
○PG（PE）→プログラマーのことで、システムに必要な機能をプログラミング言語を使って実装する職種。
○SE　→システムエンジニアのことで、クライアントにヒアリングを行い要求に最適な仕様のシステムを開発を行うこと。
○デザイナー　→クライアントや依頼者の求めに応じたデザインを考案する仕事。
○エンジニア　→工学に関する専門的な知識やスキルを持った人材のこと。
○フロントエンド・エンジニア　→フロントエンド側(Webブラウザ側)を担当するエンジニアのこと。
○サーバーサイド・エンジニア　→サーバー側で動作するプログラムの開発やデータの処理などを担当するエンジニアのこと。
○バックエンド・エンジニア　→システムやサイトを利用するユーザーの目には見えない部分を担当するエンジニア。サーバーサイドエンジニアとほとんど同じ意味で扱われるが、サーバーサイドの方がより広い範囲を表して使われる。

■UI(ユーザーインターフェース)
○UI →ユーザーと製品・サービスの接点で、ユーザーの視覚に触れる全ての情報のこと。
 ○CUI　→キーボードで入力されるコマンドによって、OS上の操作を行うUIのこと。
 ○GUI　→マウスなどのポインティングデバイスを使ってアイコンやウィンドウを操作することができるUIのこと。

■ファイル/ディレクトリ
○ファイル　→コンピュータにおけるデータの管理単位の一つで、最小の記録単位となるデータのまとまり。
○拡張子　→コンピュータで扱うファイルの種類を識別するための文字列のこと。
○ディレクトリ　→ファイルをグループ化するための特殊なファイルで、整理・管理などの目的で活用される。
　○ルート・ディレクトリ　→階層型のファイル構造における最上階層のディレクトリのこと。
　○カレント・ディレクトリ　→ユーザーが現時点で作業を行なっているディレクトリのこと。
○パス
　○絶対パス　→ルートディレクトリから目的のディレクトリまでの経路を表すパス。
　○相対パス　→ユーザーが現在作業しているディレクトリから目的のディレクトリまでの経路を表すパス。

■ネットワーク
○LAN　→家庭内や社内など限定された範囲で接続できるネットワークのこと。
○WAN　→遠く離れたエリアと繋がったネットワークのこと。
○クライアント　→何らかのサービスを提供される側のこと。
○サーバー　→他のコンピュータからのリクエストを受けて情報や処理結果を返す役割を持つコンピュータのこと。

■各種プロトコル
○HTTP　→HTMLやXMLで記述されたハイパーテキストを使ってネットワーク上で通信する仕組み。
○FTP　→ネットワーク上でファイル等の転送を行うプロトコルの一つ。
○HTTPS　→httpに暗号化技術を適用したもの。
○SSL　→インターネット上におけるウェブブラウザとウェブサーバー間での通信を暗号化し、送受信させる仕組みのこと。
○SSH　→ネットワークに接続された機器を遠隔操作し、管理するための手段。
○ポート、ポート番号　→コンピュータが通信に使用するプログラムを識別するための番号。
○HTTP、FTP、SSHの各ポート番号　→HTTPのポート番号:80番,FTPのポート番号:20番,21番,SSHのポート番号:22番
○ファイアウォール　→企業などの内部のネットワークをインターネットを通して侵入してくる不正なアクセスから守るるための防火壁のこと。
○TCP　→IPの上位プロトコルのトランスポート層で動作するプロトコル。

■ドメイン
○ドメイン　→Webサイトがどこにあるか判別するための、インターネット上の住所のこと
○DNS　→ドメインとIPアドレスを対応させて管理するシステムのこと。　
○VPN　→公衆回線をあたかも専用回線であるかのように利用できるようにするために使われる技術。
○ブリッジ　→ネットワーク上でLAN間を行き来するデータの中継機能を持った装置のこと。
○プロキシサーバー　→インターネットに直接接続できないコンピュータの代わりに、インターネットに接続してWebサイトへのアクセスを行うサーバー。

■電子メール
○SMTP　→電子メールを送信するためのプロトコル。
○POP　→電子メールを受信するためのプロトコル。
○IMAP　→メールサーバー上で電子メールを閲覧するためのプロトコル。

■セッションとクッキー
○セッション　→コンピュータネットワークで接続を確立してから切断するまでの一連の通信のこと。
○クッキー　→アクセスしてきたブラウザにWebサーバーからの情報を一時的に保存する仕組み。
○キャッシュ　→一度アクセスしたサイトのデータをブラウザで一時的に保存し、再度同じページを開いた時に表示を早くする仕組み。

■DB（データベース）について
○インデックス　→テーブルの情報を探し出す時に検索の対象としてよく使用するカラムの値だけを取り出して検索しやすいようにしておいたもの。

■テストについて
○フロー　→コンピュータプログラムを構成する命令や手続き、処理の流れのこと。

■チケット管理ツール
○タスク　→コンピュータが処理する作業の最小単位のこと。

■バージョン管理ツール
○Git　→オープンソースの分散バージョン管理システムの一つ。
○SVN　→オープンソースの集中型バージョン管理システムの一つ。
○デグレ　→新しいバージョンのソフトウェアの品質が、以前より悪くなること。
○マージツール　→ファイルの変更履歴の比較やマージ、同期など行うことができるツール
